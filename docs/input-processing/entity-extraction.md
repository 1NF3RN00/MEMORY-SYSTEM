# Entity Extraction Specification

## Overview

The middleware extracts:
structured semantic entities.

Entities support:

* retrieval precision
* metadata matching
* graph linking
* orchestration routing

Entity extraction improves:
semantic specificity.

---

# Supported Entity Categories

Examples:

* products
* services
* locations
* dates
* workflows
* plans
* organizations

Entities become:
retrieval infrastructure signals.

---

# Extraction Philosophy

Entity extraction should remain:
lightweight,
deterministic,
and inexpensive whenever possible.

The middleware avoids:
heavy model dependency
for simple extraction tasks.

high-density semantic retrieval.
