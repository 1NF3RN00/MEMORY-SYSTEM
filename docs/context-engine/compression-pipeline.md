# Adaptive Compression Extension

## Overview

Compression is:
adaptive semantic infrastructure behavior.

Different workflows require:
different levels of semantic fidelity.

The middleware dynamically selects:
compression strategies
based on:

* workflow type
* retrieval confidence
* semantic complexity
* governance requirements
* token budgets
* nuance requirements

Compression is:
context-aware and orchestration-aware.

static universal compression behavior.

Semantic fidelity requirements differ across:

* support workflows
* pricing workflows
* legal workflows
* technical workflows
* analytics workflows

Compression quality depends on:
preserving the correct level of nuance.
