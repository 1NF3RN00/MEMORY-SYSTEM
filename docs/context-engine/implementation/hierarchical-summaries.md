# Hierarchical Summary Specification

## Overview

The middleware supports:
hierarchical semantic summaries.

Hierarchical summaries allow:
multi-layer semantic compression.

Examples:

* chunk summaries
* section summaries
* document summaries
* retrieval summaries

The middleware supports:
layered semantic abstraction.

---

# Summary Hierarchy

```txt id="hs1"
Raw Content
↓
Chunk Summary
↓
Section Summary
↓
Document Summary
↓
Retrieval Summary
```

---

# Core Philosophy

The middleware prioritizes:
summary-first retrieval.

Hierarchical summaries improve:

* retrieval speed
* semantic density
* token efficiency
* context compression

Raw memory expansion occurs only when:
necessary.

---

# Summary Preservation

Summaries preserve:

* semantic meaning
* relationships
* attribution
* retrieval relevance

The middleware values:
compressed semantic continuity.
