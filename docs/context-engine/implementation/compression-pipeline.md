# Compression Pipeline Specification

## Overview

The middleware supports:
multi-stage semantic compression.

Compression exists to:
maximize semantic density.

Compression is:
relationship-aware,
governance-aware,
and token-aware.

---

# Compression Pipeline

```txt id="cp1"
Retrieved Memories
↓
Deduplication
↓
Overlap Detection
↓
Semantic Merging
↓
Summary Prioritization
↓
Detail Reduction
↓
Token Enforcement
↓
Compressed Working Context
```

---

# Compression Philosophy

Compression is:
semantic optimization infrastructure.

The middleware compresses:
redundancy,
not semantic value.

---

# Compression Levels

Supported levels:

* raw
* light
* moderate
* aggressive

Compression level selection is:
orchestration-driven.

---

# Compression Failure Handling

Poor compression quality may trigger:

* reduced compression
* alternate summaries
* raw memory expansion

Compression quality remains:
observable and measurable.
