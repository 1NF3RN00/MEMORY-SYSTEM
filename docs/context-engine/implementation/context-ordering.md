# Context Ordering Specification

## Overview

The middleware strategically orders:
final context windows.

Ordering affects:

* retrieval quality
* generation quality
* semantic coherence
* token utilization

The middleware prioritizes:
high-value context placement.

---

# Ordering Signals

Ordering may consider:

* retrieval confidence
* source authority
* semantic importance
* relationship hierarchy
* temporal relevance

Critical context appears:
earlier in working memory.

---

# Context Philosophy

The middleware does not:
randomly concatenate memories.

Context ordering is:
semantic prioritization infrastructure.

---

# Ordering Example

```txt id="co1"
High-confidence pricing summary
↓
Supporting workflow details
↓
Related contextual memories
↓
Optional expanded detail
```

The middleware values:
semantic coherence and retrieval clarity.
