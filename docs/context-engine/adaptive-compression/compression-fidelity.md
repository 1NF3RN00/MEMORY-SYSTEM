# Compression Fidelity Specification

## Overview

Compression fidelity defines:
how much semantic nuance survives compression.

Different workflows require:
different fidelity levels.

Compression fidelity is:
explicit semantic infrastructure behavior.

---

# Fidelity Levels

Examples:

```txt id="cf1"
Level 1 — Aggressive Compression
- maximum token reduction
- summary-heavy
- low nuance preservation

Level 2 — Moderate Compression
- balanced detail retention
- moderate semantic abstraction

Level 3 — Nuance Preservation
- high semantic fidelity
- preserves edge-case detail

Level 4 — Raw Context Priority
- minimal compression
- preserves original semantic structure
```

---

# Fidelity Signals

Fidelity selection may consider:

* workflow sensitivity
* ambiguity
* governance requirements
* retrieval confidence
* graph continuity
* semantic complexity

Fidelity selection remains:
deterministic and configurable.

---

# Fidelity Philosophy

Compression quality is:
not merely token reduction.

Compression quality depends on:
preserving appropriate semantic fidelity.
