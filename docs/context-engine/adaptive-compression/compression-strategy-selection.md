# Compression Strategy Selection Specification

## Overview

The middleware dynamically selects:
compression strategies.

Compression selection depends on:

* workflow type
* token budgets
* semantic complexity
* governance requirements
* nuance sensitivity

Compression strategy selection is:
orchestration infrastructure behavior.

---

# Core Philosophy

Compression strategies should adapt to:
semantic execution context.

The middleware avoids:
globally fixed compression behavior.

---

# Example Compression Strategies

Examples:

* summary-first compression
* overlap-focused compression
* hierarchy-aware compression
* nuance-preserving compression
* aggressive token minimization

Compression strategies remain:
configurable and observable.

semantic working memory quality,
