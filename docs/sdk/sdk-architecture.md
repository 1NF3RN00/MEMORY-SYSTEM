# SDK Architecture Specification

## Overview

The middleware supports:
portable SDK consumption.

SDKs expose:
middleware infrastructure capabilities.

SDKs support:

* retrieval
* ingestion
* orchestration
* graph traversal
* analytics

The SDK layer enables:
developer integration.

---

# Core Philosophy

SDKs are:
middleware consumers,
not independent intelligence systems.

The middleware remains:
the centralized semantic infrastructure layer.

---

# SDK Objectives

SDKs attempt to:

* simplify integration
* standardize middleware access
* preserve governance
* maintain observability

The middleware values:
portable semantic infrastructure.
