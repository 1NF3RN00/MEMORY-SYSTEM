# MEMORY MODEL

Memory is append-only and versioned.

Updates NEVER overwrite existing memory.

New updates create new versions.

Older memory:

* loses relevance through deterministic scoring
* may be archived
* remains observable
* remains replayable
* remains lineage-traceable

# MEMORY STRUCTURE

A Memory Object contains Memory Chunks.

Chunks are NOT independent memory entities.

Memory Objects maintain:

* source lineage
* semantic grouping
* retrieval relationships
* observability traceability

# TEMPORARY MEMORY RULES

Temporary memory:

* participates in retrieval
* participates in ranking
* participates in compression
* participates in context assembly

BUT:

* expires after query completion
* is archived for replay and analytics
* is NOT stored in operational retrieval memory
