# Adjacency Model Specification

## Overview

The middleware preserves:
semantic adjacency relationships.

Adjacency represents:
contextual continuity between memories.

Examples:

* neighboring paragraphs
* sequential transcript segments
* adjacent workflow steps
* related sections

Adjacency improves:
semantic continuity during retrieval.

---

# Core Philosophy

Semantic meaning often exists across:
neighboring memory structures.

The middleware avoids:
isolated fragment retrieval.

Adjacency enables:
controlled contextual expansion.

---

# Adjacency Relationships

Examples:

```txt id="am1"
Section A
↔
Section B
↔
Section C
```

Adjacency relationships remain:

* weighted
* governed
* traversable

---

# Adjacency Generation

Adjacency may derive from:

* document sequence
* workflow ordering
* transcript continuity
* hierarchical structure

Adjacency generation is:
deterministic-first.

# Adjacency Philosophy

Adjacency exists to:
preserve semantic continuity.

The middleware values:
relationship-aware retrieval
over isolated chunk retrieval.
