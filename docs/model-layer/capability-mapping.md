# Capability Mapping Specification

## Overview

The middleware maps:
semantic infrastructure tasks
to:
model capabilities.

Capability mapping determines:
which models support:
which infrastructure operations.

Capability mapping supports:
portable orchestration behavior.

---

# Core Philosophy

Not all models support:
identical semantic infrastructure tasks.

The middleware explicitly maps:
capabilities to providers.

---

# Example Capabilities

Examples:

* long-context generation
* embeddings
* summarization
* classification
* reranking
* structured outputs

Capability mappings remain:
configuration-driven.

---

# Capability Philosophy

Capability mapping enables:
provider-agnostic orchestration infrastructure.
