# Model Abstraction Specification

## Overview

The middleware supports:
provider-agnostic model abstraction.

Models are treated as:
replaceable semantic infrastructure utilities.

The middleware abstracts:

* providers
* APIs
* token accounting
* embedding systems
* generation systems

The middleware remains:
model-independent infrastructure.

---

# Core Philosophy

Models should not:
control middleware architecture.

The middleware owns:

* orchestration
* retrieval
* memory
* governance
* compression
* workflow execution

Models perform:
bounded semantic processing tasks.

---

# Supported Model Categories

Examples:

* generation models
* embedding models
* summarization models
* reranking models
* classification models

Model categories remain:
modular and replaceable.

---

# Abstraction Philosophy

The middleware values:
portable semantic infrastructure
over provider-specific architecture.
