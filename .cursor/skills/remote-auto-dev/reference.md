# Remote Auto Dev — Reference

## Resend setup

1. Create account at [resend.com](https://resend.com)
2. Add and verify sending domain (or use `onboarding@resend.dev` for testing)
3. Create API key → `RESEND_API_KEY=re_...`
4. Set `PERF_NOTIFY_EMAIL` to your inbox

Test without sending:

```bash
npm run perf:notify -- --wave 1 --dry-run
```

Opens `docs/performance-improvments/wave-reports/wave-1-email.html` in browser.

## Tier B — Magic link trigger API

Add to `apps/api` (or a tiny Vercel serverless function):

```
GET /perf/trigger?wave=2&token=<hmac>
```

Validation:

```typescript
import crypto from "crypto";

function validToken(wave: number, token: string, secret: string): boolean {
  const expected = crypto.createHmac("sha256", secret).update(`wave:${wave}`).digest("hex").slice(0, 32);
  return crypto.timingSafeEqual(Buffer.from(token), Buffer.from(expected));
}
```

On valid request:

1. Verify `PERF_TRIGGER_SECRET` matches
2. Queue wave run — options:
   - **GitHub Actions** `workflow_dispatch` with `wave` input (best for cloud)
   - **Webhook to always-on runner** (home PC, Railway, etc.) that runs `npm run perf:wave -- --wave N`
   - **Cursor Cloud Agent** via SDK with `cloud: { repos: [...] }` (opens PR, no local tree)

Return HTML: "Wave 2 started. You'll receive email when complete."

Token is generated by `notify-wave.mjs` using same HMAC as above.

## Tier C — Reply-to-start via Resend Inbound

1. Resend Dashboard → Domains → Enable **Inbound**
2. MX records point to Resend
3. Webhook URL: `POST /perf/inbound` on your API
4. Parse inbound payload for subject/body matching:

   ```
   START WAVE 2
   ```

5. Reject if sender ≠ `PERF_NOTIFY_EMAIL` (allowlist)
6. Call same queue logic as magic link

Reply parsing is fragile (forwards, signatures). Prefer magic links for production.

## GitHub Actions pattern (unattended waves)

```yaml
# .github/workflows/perf-wave.yml
on:
  workflow_dispatch:
    inputs:
      wave:
        type: string
        required: true

jobs:
  run:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: "22" }
      - run: npm ci && npm install -D @cursor/sdk
      - run: npm run perf:wave -- --wave ${{ inputs.wave }}
        env:
          CURSOR_API_KEY: ${{ secrets.CURSOR_API_KEY }}
          RESEND_API_KEY: ${{ secrets.RESEND_API_KEY }}
          PERF_NOTIFY_EMAIL: ${{ secrets.PERF_NOTIFY_EMAIL }}
```

Magic link can hit `workflow_dispatch` via GitHub API instead of local machine.

## Outcomes aggregation

`collect-wave-outcomes.mjs` reads each sprint's `outcomes.md` and extracts:

- Implementation / verification status
- Verification score
- Objectives met
- Short implementation summary

Extend parsers if `outcomes.md` format evolves.
